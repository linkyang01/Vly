# Vly 第一阶段 MVP 范围

## 1. MVP 目标

第一阶段只验证一件事：Vly 能否作为 macOS 本地多格式播放器成立。

不要把第一阶段做成完整 Infuse。第一阶段的成功标准是播放内核和基础控制可用。

## 2. MVP 必须包含

### 文件输入

- 通过按钮打开本地视频文件；
- 支持拖拽视频文件到窗口播放；
- 记录当前打开的文件名。

### 播放内核

- 集成 KSPlayer；
- 使用 FFmpeg 相关能力支撑多格式播放；
- 建立 PlayerCore 适配层；
- UI 不直接操作第三方播放器对象。

### 播放控制

- 播放；
- 暂停；
- 进度条；
- 当前时间；
- 总时长；
- 音量；
- 全屏；
- 基础错误提示。

### 格式验证

- MP4；
- MKV；
- H.264；
- H.265 / HEVC；
- AAC；
- AC3。

## 3. MVP 不包含

- iOS；
- iPadOS；
- tvOS；
- 媒体库；
- 电影详情；
- 海报墙；
- TMDB；
- NAS；
- SMB；
- WebDAV；
- DLNA；
- iCloud；
- 订阅或付费；
- 复杂设置；
- 多主题皮肤。

## 4. 建议页面

### 空状态

- 显示 Vly 标识；
- 显示“打开视频”按钮；
- 显示拖拽区域；
- 简要说明支持 MP4 / MKV。

### 播放状态

- 主区域显示视频；
- 底部显示控制栏；
- 显示文件名；
- 支持鼠标悬浮显示控制栏。

### 错误状态

- 显示错误原因；
- 提供重新打开文件按钮；
- 不崩溃。

## 5. 验收标准

MVP 完成必须满足：

1. 项目能编译；
2. App 能启动；
3. 能通过文件选择器打开视频；
4. 能拖拽打开视频；
5. MP4 能正常播放；
6. MKV 能正常播放；
7. 播放、暂停、进度、音量、全屏可用；
8. 播放失败时不会崩溃；
9. 代码结构能支持后续字幕和媒体库扩展；
10. README 与文档同步更新。

## 6. 进入下一阶段的条件

只有当 MVP 验收通过后，才进入阶段二：字幕、音轨、播放列表和快捷键。
