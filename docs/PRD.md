# Vly 产品需求文档

## 1. 产品定位

Vly 是一款面向 Apple 生态的全能媒体播放器，长期目标是提供类似 Infuse 的播放体验，覆盖本地文件、家庭媒体库、NAS、WebDAV、DLNA、字幕音轨、播放进度同步等场景。

Vly 的核心不是“打开一个视频文件”，而是帮助用户统一管理和播放自己的影视资源。

## 2. 目标用户

| 用户类型 | 需求 |
|---|---|
| 本地影音用户 | 播放 MKV、MP4、AVI、FLV 等多格式视频 |
| NAS 用户 | 访问 SMB / WebDAV / DLNA 媒体源 |
| 电影收藏用户 | 海报墙、电影信息、继续观看 |
| 剧集用户 | 自动识别季、集、观看进度 |
| Apple 生态用户 | Mac、iPhone、iPad、Apple TV 多端体验 |

## 3. 对标产品

| 产品 | 参考点 |
|---|---|
| Infuse | 媒体库、NAS、字幕音轨、Apple 生态体验 |
| IINA | macOS 本地播放器体验、快捷键、极简界面 |
| VLC | 格式兼容性和多协议播放能力 |

## 4. 核心价值

1. 强格式兼容：通过 KSPlayer 与 FFmpeg 能力支持多格式播放。
2. Apple 原生体验：使用 SwiftUI 构建轻量、现代的原生界面。
3. 媒体库管理：自动识别电影、电视剧、季集和观看进度。
4. 网络媒体源：支持 NAS、WebDAV、DLNA 等家庭媒体访问场景。
5. 跨设备同步：后续通过 iCloud 同步播放进度和偏好。

## 5. 阶段目标

### 阶段一：macOS 最小播放器 Demo

目标：验证播放器内核能稳定播放本地视频。

必须支持：

- 打开本地视频；
- 拖拽播放；
- MP4 与 MKV 播放验证；
- 播放、暂停、进度、音量、全屏；
- 基础错误提示。

### 阶段二：播放器体验增强

- 播放列表；
- 最近打开；
- 快捷键；
- 外挂字幕；
- 字幕选择；
- 音轨选择。

### 阶段三：本地媒体库

- 文件夹扫描；
- 电影与剧集识别；
- 海报墙；
- 继续观看；
- 播放历史；
- 搜索和筛选。

### 阶段四：网络媒体源

- SMB；
- WebDAV；
- DLNA；
- HTTP URL 播放；
- 网络字幕加载。

### 阶段五：Apple 生态扩展

- iCloud 同步；
- iOS / iPadOS；
- tvOS；
- 多端继续观看。

## 6. 非目标范围

第一阶段不做：

- iOS / iPadOS / tvOS；
- 媒体库；
- 海报墙；
- TMDB；
- NAS；
- iCloud；
- 商业化；
- 复杂设置中心。

## 7. 成功标准

Vly 第一阶段成功的标准不是界面多漂亮，而是：

1. 能用 KSPlayer 播放本地视频；
2. MP4 与 MKV 均能稳定播放；
3. 基础控制完整可用；
4. 工程结构清晰；
5. 后续可以自然扩展字幕、音轨和媒体库。
