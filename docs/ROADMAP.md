# Vly 开发路线图

## 阶段 0：项目重启与需求冻结

目标：完成项目重新定位，不继续修补旧结构。

交付物：

- README.md；
- AGENTS.md；
- docs/PRD.md；
- docs/TECHNICAL_ARCHITECTURE.md；
- docs/FORMAT_SUPPORT.md；
- docs/MVP_SCOPE.md。

验收标准：

- 产品定位明确；
- 第一阶段范围明确；
- 不再同时展开多端和媒体库；
- 播放内核路线明确。

## 阶段 1：macOS 最小播放器 Demo

目标：验证 KSPlayer + FFmpeg 能力是否可支撑 Vly。

功能：

- 创建 macOS SwiftUI 工程；
- 集成 KSPlayer；
- 建立 PlayerCore 适配层；
- 打开本地文件；
- 拖拽播放；
- 播放 / 暂停 / 进度 / 音量 / 全屏；
- MP4 与 MKV 播放验证。

验收标准：

- 项目可编译；
- MP4 可播放；
- MKV 可播放；
- 基础控制可用；
- 播放失败有错误提示。

## 阶段 2：播放器体验增强

目标：从 Demo 变成日常可用播放器。

功能：

- 播放队列；
- 最近打开；
- 快捷键；
- 记忆上次播放进度；
- 基础设置；
- 更完整的错误提示。

## 阶段 3：字幕与音轨

目标：建立播放器差异化能力。

功能：

- 自动加载同名 SRT；
- 手动加载字幕；
- 字幕开关；
- 字幕延迟调整；
- 内嵌字幕识别；
- 多音轨切换；
- 字幕样式设置。

## 阶段 4：本地媒体库

目标：开始接近 Infuse 类体验。

功能：

- 添加本地媒体文件夹；
- 扫描电影与剧集；
- 识别文件名；
- 海报墙；
- 详情页；
- 继续观看；
- 最近添加；
- 搜索与筛选。

## 阶段 5：元数据服务

目标：完善影视资料展示。

功能：

- TMDB 接入；
- 海报缓存；
- 背景图缓存；
- 电影简介；
- 演员信息；
- 剧集季集识别；
- 手动匹配与纠错。

## 阶段 6：网络媒体源

目标：支持家庭影音库。

功能：

- SMB；
- WebDAV；
- DLNA / UPnP；
- HTTP URL；
- 网络源收藏；
- 网络文件播放进度记录。

## 阶段 7：Apple 生态同步

目标：建立多端体验基础。

功能：

- iCloud 同步播放进度；
- iCloud 同步偏好；
- iCloud 同步媒体库索引；
- 多端继续观看。

## 阶段 8：多端扩展

目标：扩展到 iOS、iPadOS、tvOS。

优先级：

1. iPadOS；
2. iOS；
3. tvOS。

## 当前推荐执行顺序

当前只执行阶段 0 和阶段 1。

不要提前做：

- iOS；
- 媒体库；
- 海报墙；
- NAS；
- iCloud。
